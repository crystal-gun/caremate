# CareMate (케어메이트)
## 제품 요구사항 문서 (PRD) v2.0

---

## 1. 제품 비전

> "버튼 하나로 오늘의 루틴을 기록하면, AI가 바로 반응합니다."

**앱 이름: CareMate / 케어메이트**

CareMate v1은 **건강 루틴 관리 + Daily Care Comment + 무료 건강 리포트 + 동의 기반 보장 점검 상담 연결 시스템**입니다.

혼자 기록하다 멈추는 사람을, 반응받는 경험으로 전환한다. 완전 무료이며, 루틴 하나만 입력해도 앱이 즉시 작동한다.

**이 PRD에서 사용 금지 표현:** "AI 주치의" (의료 행위 암시), "닥터민", "CateMate", "DB 판매", "보험 DB"

---

## 2. 문제 정의

건강에 관심 높은 3040은 많다. 부모님의 병을 보며 루틴을 시작하는 사람도 많다. 하지만 대부분의 건강 앱에서 기록하면 그래프가 쌓일 뿐이다. 스킵해도 아무도 알아채지 않는다. 반응이 없는 기록은 오래 지속되지 않는다.

**핵심 문제:** 건강 루틴 지속 실패의 원인은 의지 부족이 아니라, 아무도 반응해주지 않기 때문이다.

---

## 3. 목표 사용자

**핵심 타겟:** 부모님의 병을 보며 건강 루틴을 시작했지만, 혼자 기록하는 앱에는 오래 머물지 못했던 3040 건강관리 관심층.

- 영양제를 챙기거나 운동 목표를 세웠고, 한 주는 열심히 했지만 두 번째 주부터 흐지부지된 경험이 있다.
- 복잡한 건강 정보 입력보다 "지금 바로 시작"을 원한다.
- 건강 앱을 깔았다가 지워본 경험이 있다.

---

## 4. 핵심 원칙: Progressive Onboarding

> "최소 입력으로 바로 작동하고, 정보가 쌓일수록 더 똑똑해진다."

CareMate는 건강 정보를 모두 입력해야 작동하는 앱이 아니다. 3단계 온보딩은 각각 독립적으로 완료 가능하며, 상위 단계일수록 더 풍부한 리포트를 제공하는 인센티브 구조다.

| 단계 | 이름 | 필수 여부 | 완료 시 보상 |
|------|------|-----------|------------|
| 1 | Quick Start | 필수 | 즉시 체크인 시작 가능 |
| 2 | Health Context | 선택 | 미니 건강 리포트 즉시 생성 |
| 3 | Report Enrichment | 선택 | 정식 리포트 품질 향상 + 상담 CTA 활성화 |

모든 건강 관련 DB 필드는 nullable. `profile_completion_score`(0-100)로 입력 수준 추적.

## 5. 목표 및 성공 지표

### 제품 목표 (베타/MVP 검증 목표)

| 지표 | 목표 |
|------|------|
| D30 리텐션 | 35% 이상 |
| 4주 사용자 체크인 완료율 | 60% 이상 |
| 앱 설치 → 정식 리포트 도달 비율 | 20% 이상 |
| 정식 리포트 열람 후 상담 신청 전환율 | 10% 이상 |
| 상담 리드 월 생성 건수 | 100건 이상 (성장 궤도 기준) |

### 비즈니스 목표

- v1 수익 검증: 동의 기반 상담 리드 배정(CPA)의 단가·전환율 데이터 확보
- v1.5 진입 기준: 리드→상담 완료 전환율 30% 이상 확인 시 구독 플랜 도입 실험 착수

### 역지표 (Counter-metrics)

- Daily Care Comment 무반응률(코멘트 미열람) 50% 초과 시 콘텐츠 품질 점검
- 상담 신청 후 24시간 내 연락 미이행 건수 0 목표
- AI 코멘트 사용자 불만 신고(의료 판단 포함 등) 건당 즉시 검토

## 6. 사용자 여정

### UJ-1: Quick Start 온보딩 → 첫 체크인

**페르소나:** 39세 직장인, 비타민C를 챙기려는 결심

1. 앱 설치 → 닉네임 입력 → 루틴 1개 설정("비타민C 매일 아침 1회") → 알림 시간 설정
2. 3분 이내 설정 완료, 앱 즉시 작동
3. 알림 수신 → 완료 ✅ 탭 → Daily Care Comment 즉시 표시
4. "오늘 첫 체크인이에요. 이게 시작이에요." — 가볍지만 반응받는 경험

**성공 기준:** 설치 후 3분 내 첫 체크인 완료

---

### UJ-2: Health Context 입력 → 미니 건강 리포트

**페르소나:** 43세 여성, 아버지가 심혈관 질환

1. 홈에서 "건강 정보 추가하면 리포트 드려요" 배너 클릭
2. 건강 관심사(심혈관) → 가족력 입력 → 현재 복용 영양제 → 보험 보장 인지 여부 → 동의 완료
3. 미니 건강 리포트 즉시 생성: "아버지 심혈관 가족력 기반으로 오메가3·마그네슘 관심 있으실 것 같아요. 이번 주 루틴 기록부터 시작해봐요."
4. 계속 체크인 동기 부여

---

### UJ-3: 일상 체크인 루프 (핵심 리텐션 루프)

**페르소나:** 동일 사용자, 4주째 루틴 유지 중

- 알림 → 완료 ✅ → Daily Care Comment: "이번 주 4일 완료. 완벽하진 않지만 손 놓은 주는 아니에요."
- 알림 → 이따가 ⏰ → 30분 후 재알림 → 완료 ✅ → "결국 챙겼네요. 그게 중요한 거예요."
- 알림 → 오늘 스킵 ❌ → Daily Care Comment: "오늘은 쉬어요. 내일 다시 한 번이면 돼요."

---

### UJ-4: 주간 브리핑

**페르소나:** 동일 사용자

1. 일요일 밤 알림: "이번 주 건강 브리핑 도착"
2. "비타민C 7일 중 5일 완료. 나머지 2일, 아직 포기하지 않은 거잖아요. 다음 주 목표: 6일."
3. 운동 루틴 달성률 요약 + 다음 주 목표 1개

---

### UJ-5: 정식 건강 리포트 → 상담 신청 (핵심 수익 여정)

**페르소나:** 동일 사용자, 2주 경과 + 5회 체크인 달성

1. 트리거 충족 → 푸시 "건강 리포트 준비됐어요"
2. 리포트 열람: 체크인 패턴 분석 + Health Context 기반 AI 분석 + "아버지 심혈관 가족력 기반으로..."
3. 리포트 하단 CTA: "무료 보장 점검 상담 신청" (동의 기반 조건부 노출)
4. 상담 신청 → 제3자 제공 동의 확인 → 신청 완료
5. 관리자 리드 목록에 '신규' 등록

---

### UJ-6: 관리자 리드 관리

1. 관리자 로그인(별도 admin 경로)
2. 리드 목록(신청일·이름·연락처·상태) 열람
3. 리드 상세 확인(동의 항목·건강 관심사)
4. 상태 변경: 신규 → 연락 예정 → 연락 완료 → 상담 진행 → 상담 완료 → 계약 전환 / 보류

### F5. Daily Care Comment

Daily Care Comment는 CareMate의 핵심 차별점이다. 체크인 직후 즉각 반응해 "반응받는 경험"을 만든다.

| ID | 기능 | 설명 |
|----|------|------|
| FR-5.1 | 체크인 즉시 코멘트 생성 | 체크인(완료·이따가·스킵) 직후 Claude API로 코멘트 생성·표시; 지연 없음 |
| FR-5.2 | 상태별 톤 분기 | 완료→칭찬형, 미루기 반복→살짝 찌르는형, 스킵→공감+다음 행동 제안형, 운동 완료→응원형, 운동 패스→공감형 |
| FR-5.3 | CareMate 보이스 톤 적용 | 팩트 정확·사람 말투·살짝 웃김·의학적 단정 피하기·혼냄 없음·다음 행동 제안으로 마무리 |
| FR-5.4 | 코멘트 히스토리 저장 | daily_care_comments 테이블; 사용자가 이전 코멘트 열람 가능 |

| FR-9.4 | 리드 상태 7단계 변경 | 신규 → 연락 예정 → 연락 완료 → 상담 진행 → 상담 완료 → 계약 전환 / 보류 |
