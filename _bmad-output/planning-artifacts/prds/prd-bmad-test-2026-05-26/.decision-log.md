# 의사결정 로그 — CareMate(케어메이트) PRD

_이 파일은 PRD 작성 과정의 모든 결정·변경·가정을 기록합니다._

---

## 2026-05-26

### DL-001: Fast Path 작업 방식 선택
- **결정**: Fast Path로 PRD 초안 작성
- **이유**: 브레인스토밍 파일에 65개 아이디어와 충분한 맥락이 있어 빠른 초안 작성 가능
- **영향**: [ASSUMPTION] 태그로 불명확한 항목 표시; 이후 검토에서 해소 예정

### DL-002: 제품 범위 — 영양제 + 보험 + 응급의 통합
- **결정**: 단일 앱 내에 영양제 추적·과학 추천·보험 관리·응급 기능을 모두 포함
- **이유**: 브레인스토밍에서 "세상에 없는 조합"으로 차별화 포인트 명시됨
- **리스크**: 범위가 너무 넓어 MVP 집중도 저하 가능 → Phase별 출시로 완화

### DL-003: MVP 범위 선정
- **결정**: 복용 기록 → 그래프 → 주간 브리핑 핵심 루프 + 응급 기능을 MVP로 설정
- **이유**: 브레인스토밍 TOP 3 (건강기록부 그래프, 과학 추천, 응급 기능) 기반
- **제외 항목**: 유전자 연동, 스마트워치, 학습 AI → Phase 2/3

### DL-004: 비즈니스 모델 — 월 1,000원 구독
- **결정**: 브레인스토밍에서 합의된 월 1,000원 구독 모델 채택
- **가정 포함**: 연간 10,000원, 14일 무료 체험 (미확인)
- **보류**: 제휴 추천 수수료 모델은 공정성 우려로 초기 배제

### DL-005: 응급 기능 범위 재정의
- **결정**: 증상 기반 긴급도 판단 기능 제거; "심정지 N초 감지 → 지인/119 자동 연락"으로 단순화
- **이유**: crystal님 직접 요청 — 법적 복잡성 없이 실질적 안전망 역할에 집중
- **영향**: 스마트워치 연동이 응급 기능의 필수 전제 조건이 됨 (Phase 2 → MVP로 이동)
- **제거된 항목**: FR-9.1(증상 긴급도 판단), FR-9.2(위치 기반 병원 안내), FR-9.3(119 원터치), FR-9.5(야간 병원 안내)
- **OQ 변경**: OQ-2(의료기기 여부) → 심정지 임계값 결정으로 대체; OQ-6(면책 범위) → 워치 API 제약 확인으로 대체

### DL-006: ConsumerLab 등 외부 데이터 라이선스 계약 불가 확정
- **결정**: ConsumerLab, NSF International 등 독립 검증 기관 데이터 라이선스 계약 불가
- **영향**: FR-4.1 "과학 기반 랭킹"의 데이터 소스를 자체 큐레이션으로 전환
- **채택 방식**: PubMed·FDA·EFSA·식약처 공개 데이터 자체 큐레이션 + 영양학·약학과 대학 연구실 무상 자문 파트너십(논문 공동 크레딧 제공) 병행
- **비용**: 데이터 소스 모두 무료 공개 데이터; 전문가 자문은 논문 크레딧·앱 노출로 대체하여 무상
- **OQ-4 해소**

### DL-007: [ASSUMPTION] 앱 이름 — 가칭 닥터민(DoctorMin)
- **결정**: PRD 작성 편의상 가칭 사용; 브랜딩은 별도 결정 필요
- **미결**: OQ-1

### DL-008: Finalize 검토 결과 반영
- **119 자동 연락 기술적 불가 → B안 채택**: iOS/Android 앱의 자동 전화 발신 불가 → 지인 SMS·푸시 자동 발송 + 119 원터치 버튼으로 대체
- **바이오리듬 브랜드 충돌 → B안 채택**: 과학 추천과 시각 분리된 엔터테인먼트 기능으로 명시; 의학적 근거 없음 표시
- **추가 반영**: 주치의 반말 톤 FR-1.6 신설, 비급여 커버리지 FR-8.6 추가, 미루기→그래프 연동 명세, NON-GOAL 섹션 추가, Phase 3 zombie ID 정리

## 2026-05-28 — v2.0 전면 수정 (Product Direction v1.0 기준)

**트리거:** `/bmad-prd` 호출. Product Direction v1.0(2026-05-28) + PRFAQ Distillate + Brief v2.0 기준 전면 수정.

### DL-010: 제품 정체성 전면 전환 (v1.1 내용 무효화)

- **결정:** "AI 영양제 추천 앱 / AI 주치의" → "건강 루틴 관리 + Daily Care Comment + 무료 건강 리포트 + 동의 기반 상담 리드 CPA"
- **이유:** Product Direction v1.0 §3에서 확정. 기존 정체성은 의료 행위 규제 리스크 + v1 기능 과부하
- **영향:** 9개 기능 그룹(F1-F11) 중 F1(주치의 코어)·F4(영양제 추천)·F5(랭킹)·F9(SOS)·F10(재고)·F11(병원 연동) 제거 또는 재편

### DL-011: 기능 그룹 재편

| 구 기능 그룹 | 처리 |
|------------|------|
| F1 AI 주치의 코어 (대화형 UI) | 제거 — Daily Care Comment(새 F5)로 대체; 대화 인터페이스 v1 미포함 |
| F2 온보딩 & 건강 프로필 | → 새 F2 Progressive Onboarding (3단계 구조로 재설계) |

| F6 운동 기록 | 축소 → F3에 통합 (운동 약속 루틴 체크 방식만) |

### DL-012: Daily Care Comment 독립 기능 그룹으로 승격

### DL-013: 비즈니스 모델 전환 — 구독 제거

- **결정:** 4단계 구독(DL-004) 전면 취소 → 완전 무료 + CPA 리드 배정
- **이유:** v1은 수익 검증 단계. 포트원 결제·Feature Gating 미구현
- **금지 표현:** "DB 판매", "보험 DB" → PRD 전체에서 제거

### DL-014: 운동 기능 범위 축소

- **결정:** FR-6.1(종류·강도·칼로리 기록) 제거 → 운동 약속 루틴 체크 방식만 (FR-3.2)
- **이유:** Product Direction v1.0 §6, Brief v2.0 확정 사항

### DL-015: 건강 리포트 2단계 구조 확정

- **결정:** 미니 리포트(Health Context 완료 직후) + 정식 리포트(14일 AND 5회 체크인)
- **이유:** Progressive Onboarding 인센티브 + 리텐션 기간 확보
