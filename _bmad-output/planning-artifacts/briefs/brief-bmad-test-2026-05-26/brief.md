# 제품 브리프: CareMate (케어메이트)

---

## 요약

건강 루틴을 시작하는 사람은 많다. 오래 유지하는 사람은 드물다. 문제는 의지가 아니다 — 아무도 반응해주지 않기 때문이다.

케어메이트는 버튼 하나로 오늘의 루틴을 기록하면 AI가 즉시 반응하는 앱이다. 혼자 기록하다 멈추는 사람을, 반응받는 경험으로 전환한다. 완전 무료이며, 2주 이상 기록이 쌓이면 무료 건강 리포트를 제공한다. 리포트를 본 뒤 보장 점검 상담이 궁금한 사용자가 동의한 경우에만 상담으로 연결한다.

---

## 문제

부모님의 병을 보며 "나는 달라야지"를 결심하는 3040은 많다. 하지만 대부분의 건강 앱은 기록하면 그래프가 쌓일 뿐이다. 스킵해도 아무도 알아채지 않는다. 결국 앱은 홈 화면 어딘가로 사라진다.

반응이 없는 기록은 오래 지속되지 않는다.

---

## 대상 사용자

**핵심 타겟:** 부모님의 병을 보며 건강 루틴을 시작했지만, 혼자 기록하는 앱에는 오래 머물지 못했던 3040 건강관리 관심층.

영양제를 챙기거나 운동 목표를 세웠고, 한 주는 열심히 했지만 두 번째 주부터 흐지부지된 경험이 있다. 복잡한 입력보다 "지금 바로 시작"을 원한다.

---

## 솔루션

**CareMate v1은 건강 루틴 관리 + Daily Care Comment + 무료 건강 리포트 + 동의 기반 보장 점검 상담 연결 시스템입니다.** "AI 영양제 추천 앱"이 아닙니다.

### 핵심 차별점: Daily Care Comment

체크인(완료 ✅ / 이따가 ⏰ / 오늘 스킵 ❌) 직후 AI가 즉각 반응한다. 잘 지켰을 때는 가볍게 칭찬하고, 사흘째 미루고 있으면 살짝 찌르며 다음 행동을 제안한다. 빠진 날에는 혼내지 않고 내일 다시 시작할 수 있게 돕는다. 알림에서 버튼을 누르면 앱을 열지 않아도 체크인이 완료된다.

### Progressive Onboarding 원칙

> "최소 입력으로 바로 작동하고, 정보가 쌓일수록 더 똑똑해진다."

- **1단계 Quick Start (필수):** 닉네임 + 루틴 1개만 입력하면 즉시 시작. 건강검진 결과·가족력 없어도 된다.
- **2단계 Health Context (선택):** 건강 관심사·가족력·복용 영양제 입력 시 미니 건강 리포트 즉시 생성.
- **3단계 Report Enrichment (선택):** 건강검진 내역 추가 시 정식 리포트 품질 향상.

### 건강 리포트 (2단계)

- **미니 리포트:** Health Context 완료 직후 즉시 생성. 입력 인센티브.
- **정식 리포트:** 14일 경과 **AND** 체크인 5회 이상 달성 시 자동 생성. 리포트 하단에 보장 점검 상담 신청 CTA 노출.

### 운동 기능

운동 종류·강도·칼로리 상세 기록이 아니다. 운동 약속 루틴을 만들고 (예: 매주 수/목 30분 운동), 알림 수신 후 완료/미루기/오늘 패스로 체크한다. Daily Care Comment가 동일하게 반응한다.

### 주간 브리핑

매주 자동 생성. 복용률·운동 달성률 요약 + AI 위트 총평 + 다음 주 목표 제안 1개.

---

## 차별화

| 항목 | 기존 건강 앱 | CareMate |
|------|-------------|----------|
| 체크인 후 반응 | 없음 (그래프만 쌓임) | Daily Care Comment 즉각 생성 |
| 진입 장벽 | 건강 정보 입력 후 시작 | 루틴 1개로 즉시 시작 |
| 동기부여 방식 | 알림·경고·배지 | 사람 같은 반응, 혼냄 없음 |
| 수익 구조 | 구독 또는 광고 | 동의 기반 상담 리드 CPA, 사용자는 무료 |

진짜 장벽은 데이터 축적이다. 쌓인 루틴 기록과 AI가 파악한 패턴은 다른 앱으로 옮겨갈 수 없다.

---

## 비즈니스 모델

**v1 완전 무료. 결제·구독 없음.**

수익은 동의 기반 상담 리드 배정(CPA)으로 검증한다. 정식 리포트를 열람한 사용자가 자발적으로 보장 점검 상담을 신청하고, 제3자 제공 동의를 완료한 경우에만 상담사에게 리드를 연결한다. 리드당 수수료 또는 계약 성과 기반으로 정산.

"DB 판매", "보험 DB", "고객 DB 수집" 표현 사용 금지. "동의 기반 상담 리드 배정"으로 표현.

v1 리드 전환 데이터를 기반으로 v1.5에서 구독 플랜 도입 여부를 결정.

---

## v1 기능 범위

**포함:**
- 이메일·카카오 로그인, 회원 탈퇴 및 데이터 삭제 (PIPA 30일)
- 영양제 복용 루틴 + 운동 약속 루틴 설정 (복수 루틴 지원, routine_schedules 분리)
- 체크인 UX: 완료 / 이따가 / 오늘 스킵 — 알림에서 직접
- Daily Care Comment (Claude API, 체크인 직후 즉각 반응)
- 주간 브리핑 (자동 생성)
- 건강 리포트 2단계 (미니 리포트 / 정식 리포트)
- 동의 관리 4종 분리 (consents append-only)
- 상담 신청 폼 + 관리자 리드 목록 (7단계 상태 관리)

**v1 제외:**
- 영양제 AI 추천, 영양제 랭킹·성분 비교 (v3)
- 스마트워치 SOS (v2 별도 검토)
- 건강검진 OCR 자동 파싱 (v2)
- 유전자검사 실제 분석 (v2)
- 포트원 결제·구독 4단계 (v1.5 이후)

---

## 로드맵

**v2:** 유전자검사 신청·결과 기반 맞춤 건강 리포트, 건강검진 OCR, 스마트워치 SOS 기술 검토, 구독 플랜 도입 여부 결정.

**v3:** 영양제 랭킹·성분 비교·리뷰·커머스, 보험 청구 자동화.

## 성공 기준 (v1)

- D30 리텐션 35% 이상
- 4주 사용자 체크인 완료율 60% 이상
- 정식 리포트 열람 후 상담 신청 전환율 10% 이상
- 상담 리드 월 100건 이상
