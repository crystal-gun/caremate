# CareMate Product Direction v1.0

> 이 문서는 CareMate v1 MVP부터 v3까지의 제품 방향을 확정한 최상위 기준문서입니다.
> 이후 PRFAQ, Brief, PRD, Architecture, DB Schema, Epics는 이 문서를 기준으로 수정합니다.
> 이 문서와 충돌하는 기존 문서의 내용은 이 문서가 우선합니다.

---

## 1. 문서 목적

이 문서는 CareMate 프로젝트의 제품 방향을 재정렬하고, 기존 문서(PRD, Architecture, Epics)에 산재된 방향 충돌을 해소하기 위해 작성된 최상위 기준문서입니다.

브레인스토밍 세션(2026-05-28)에서 확정된 결정사항을 집대성하며, 개발팀·기획팀·이해관계자 모두가 동일한 기준으로 제품을 이해할 수 있도록 합니다.

---

## 2. 앱명 및 제품 한 줄 정의

**앱명:** CareMate / 케어메이트

**사용 금지 명칭:** CateMate, 닥터민 (기존 문서에서 완전 제거)

**제품 한 줄 정의:**
> 최소 입력으로 바로 작동하는 건강 루틴 관리 앱. 쌓인 데이터로 무료 건강 리포트를 제공하고, 동의한 사용자를 보장 점검 상담으로 자연스럽게 연결한다.

---

## 3. 제품 정체성

### v1 MVP 정체성

**CareMate v1은 "AI 영양제 추천 앱"이 아닙니다.**

CareMate v1은 **"건강 루틴 관리 + 무료 건강 리포트 + 동의 기반 상담 리드 생성 시스템"**입니다.

### 차별점

CareMate의 핵심 차별점은 **보살핌 UX**입니다.

사용자가 앱에 들어와 지루하게 입력하는 앱이 아니라:
- 푸시 알림에서 버튼 하나로 완료/미루기/스킵을 기록하고
- AI가 매일 짧게 반응해주는 경험

"기록했다"가 아니라 **"반응을 받았다"**고 느끼게 하는 것이 핵심입니다.

### 기존 방향과의 차이

| 항목 | 기존 방향 (변경 전) | 새 방향 (이 문서 기준) |
|------|------------------|----------------------|
| MVP 정체성 | AI 영양제 추천 앱 | 건강 루틴 관리 + 리포트 + 상담 리드 |
| 운동 기능 | 종류·강도·칼로리 상세 기록 | 루틴 약속 관리 (목표 설정 + 완료/미루기/패스) |
| Daily 반응 | 주간 브리핑만 | Daily Care Comment 필수 + 주간 브리핑 |
| 수익 모델 | 포트원 4단계 구독 결제 | 동의 기반 상담 리드 배정 (CPA) |
| 영양제 랭킹 | v1 MVP 포함 | v3으로 이동 |
| 스마트워치 SOS | Phase 2 포함 | v2 이후 별도 검토 항목으로 보류 |

## 4. v1 MVP 목표

### 제품 목표

1. **루틴 정착률**: 4주 사용자의 체크인 완료율 60% 이상
2. **리포트 도달률**: 가입 후 14일 AND 5회 체크인 달성 사용자 비율 50% 이상
3. **상담 전환율**: 정식 리포트 열람 후 상담 신청 전환율 10% 이상

### 비즈니스 목표

- v1 수익 검증 수단: 동의 기반 보장 점검 상담 리드 생성
- 유료 구독·결제 없이 리드 품질과 전환율로 비즈니스 모델 검증
- 이후 v1.5에서 구독 플랜 도입 여부를 리드 전환 데이터 기반으로 판단

### 성공 기준

- 상담 리드 월 100건 이상 생성
- 리드→상담 완료 전환율 30% 이상
- D30 리텐션 35% 이상

## 5. v1 핵심 사용자 여정

### UJ-1: Quick Start 온보딩 → 첫 체크인

이름/닉네임 → 루틴 1개 설정(영양제 or 운동) → 알림 시간 → 앱 즉시 작동.

첫 알림 수신 → 완료/미루기/스킵 → Daily Care Comment 즉시 표시.

**원칙:** 가입 후 3분 내 첫 체크인 경험 완료. 가족력·보험 정보 없어도 작동.

---

### UJ-2: Health Context 입력 → 미니 건강 리포트

건강 관심사 → 가족력(선택) → 복용 영양제/약(선택) → 보험 보장 인지 여부 → 동의 관리(4종 분리) → **미니 건강 리포트 즉시 생성**.

**원칙:** 입력이 늘수록 즉각 보상(리포트). 강제가 아닌 인센티브.

---

### UJ-3: 일상 체크인 루프 (핵심 리텐션 루프)

**복용 루틴:** 알림 → 완료 ✅ / 이따가 ⏰(재알림 시간 설정) / 오늘 스킵 ❌ → Daily Care Comment 즉시 표시.

**운동 루틴:** 운동 시작 알림 → 종료 예상 시간 알림 → 완료 ✅ / 미루기 ⏰ / 오늘 패스 ❌ → Daily Care Comment 즉시 표시.

Daily Care Comment 톤 분기:
- 완료 → 칭찬형
- 미루기 반복 → 살짝 찌르는 형
- 스킵 → 공감 + 다음 행동 제안형

---

### UJ-4: 주간 브리핑

매주 일요일 밤(또는 사용자 설정 시간) 자동 발송.

복용률(완료/미루기/스킵 구분) + 운동 루틴 달성 현황 + AI 위트 코멘트 + 다음 주 목표 제안 1개.

---

### UJ-5: 정식 건강 리포트 → 상담 신청 전환 (핵심 수익 여정)

**트리거:** 가입 후 최소 14일 경과 **AND** 체크인 합산 5회 이상.

미충족 시: "리포트 준비 중" 상태 + 추가 체크인 유도 메시지.

트리거 충족 → 푸시 "건강 리포트 준비됐어요" → 리포트 열람(복용 패턴 + 운동 현황 + Health Context 기반 AI 분석) → **리포트 하단 CTA "무료 보장 점검 상담 신청"** → 동의 확인 → 신청 완료 → 관리자 리드 목록 '신규' 등록.

---

### UJ-6: 관리자 상담 리드 관리

관리자 로그인(별도 admin 경로) → 리드 목록(신청일/이름/연락처/상태) → 리드 상세(신청 내용/동의 항목/건강 관심사) → 상태 변경.

상태값: 신규 / 연락 예정 / 연락 완료 / 상담 진행 / 상담 완료 / 계약 전환 / 보류

## 6. v1 핵심 기능 범위

### F1. 인증 & 사용자 프로필
- 이메일/비밀번호 가입·로그인 ✅
- 카카오 소셜 로그인 ✅
- 애플 로그인 🔶 (iOS 앱스토어 제출 시 필수, 웹만이면 선택)
- 닉네임·프로필 기본 정보 ✅
- 회원 탈퇴 및 데이터 삭제 (PIPA 30일 이내) ✅

### F2. Progressive Onboarding
- Quick Start: 루틴 1개만으로 즉시 시작 ✅
- Health Context 입력 (선택): 건강 관심사·가족력·복용 영양제·보험 보장 인지 ✅
- Report Enrichment 입력 (선택): 건강검진 내역·상세 병력·상담 관심 영역 ✅
- `profile_completion_score` 추적 (리포트 품질 및 CTA 노출 기준) ✅
- 온보딩 이후 언제든 추가 입력 가능 ✅

### F3. 루틴 설정 & 알림
- 영양제 복용 루틴 생성: 이름·시간·횟수·복용 조건 ✅
- 운동 목표 루틴 생성: 요일·시간·목표 시간 (종류·강도·칼로리 없음) ✅
- 루틴 수정·삭제·일시 중지 ✅
- 푸시 알림: 복용 시작, 운동 시작, 운동 종료 예상 (FCM/APNs) ✅
- 이따가 → 재알림 시간 설정 ✅
- 루틴 복수 설정 지원 ✅

### F4. 체크인 & 기록
- 완료 / 미루기 / 오늘 스킵 버튼 (알림에서 바로 가능) ✅
- 체크인 기록 저장 (날짜·루틴·상태·시간) ✅
- 체크인 히스토리 달력 뷰 ✅
- 복용률·운동 달성률 그래프 (주간·월간) ✅
- 미루기 → 재알림 연동 ✅

### F5. Daily Care Comment
- 체크인 직후 AI 코멘트 즉시 생성·표시 (Claude API) ✅
- 상태별 톤 분기 (완료·미루기·스킵·운동 완료·패스) ✅
- CareMate 보이스 톤 적용 ✅
- 코멘트 히스토리 저장 ✅
- 하루 마감 총평 ❌ (v1.5 이후)

### F6. 주간 브리핑
- 매주 자동 생성 (Railway Cron 스케줄러) ✅
- 복용률·운동 달성률 요약 + AI 위트 총평 + 다음 주 목표 제안 ✅
- 브리핑 히스토리 보관 ✅

### F7. 건강 리포트
- 미니 리포트: Health Context 완료 직후 즉시 생성 ✅
- 정식 리포트 트리거: 14일 경과 AND 5회 체크인 ✅
- "리포트 준비 중" 상태 + 체크인 유도 (트리거 미충족 시) ✅
- 정식 리포트: 체크인 패턴 + Health Context 기반 AI 생성 ✅
- 리포트 하단 상담 신청 CTA (동의 기반 조건부 노출) ✅

### F8. 동의 관리
- 개인정보 수집·이용 동의 (필수) ✅
- 민감정보 처리 동의 (건강 정보 입력 전 별도 동의) ✅
- 제3자 제공 동의 (상담 연결 전 필수) ✅
- 마케팅 수신 동의 (선택) ✅
- 동의 이력 저장 (날짜·버전·항목 — 법적 증빙) ✅
- 동의 철회 기능 ✅

### F9. 상담 신청 & 관리자 리드 관리
- 보장 점검 상담 신청 폼 (이름·연락처·관심 영역) ✅
- 제3자 제공 동의 확인 후 신청 처리 ✅
- 관리자 리드 목록 열람 + 상태 변경 (7단계) ✅
- 관리자 메모·담당자 배정·필터·통계 ❌ (v1.5~v2)

## 9. 비즈니스 모델

### v1 수익 구조

**동의 기반 상담 리드 배정 (CPA 모델)**

- 사용자가 건강 리포트를 받은 후 자발적으로 보장 점검 상담을 신청
- 제3자 제공 동의를 포함한 명시적 동의 완료 후에만 리드 생성
- 리드를 보험설계사/FA/내부 직원에게 배정
- 리드당 수수료 또는 계약 성과 기반으로 정산

**공식 표현 원칙:**
- 사용: "동의 기반 상담 리드 배정", "보장 점검 상담 신청 고객 연결"
- 금지: "DB 판매", "보험 DB", "고객 DB 수집"

### 수익 검증 지표 (v1)

- 상담 리드 월 생성 건수
- 리드→상담 완료 전환율
- 리드→계약 전환율 (CPA 정산 기준)

## 10. 보이스 톤 원칙

CareMate의 AI(Daily Care Comment, 주간 브리핑, 건강 리포트)는 다음 톤을 유지합니다.

| 원칙 | 설명 |
|------|------|
| 팩트는 정확하게 | 복용률, 체크인 횟수 등 데이터는 정확히 |
| 말투는 사람처럼 | 딱딱한 시스템 언어 금지 |
| 살짝 웃기게 | 유머는 있되 과하지 않게 |
| 너무 장난스럽지 않게 | 건강 정보의 신뢰도 유지 |
| 의학적 단정 피하기 | "~에 좋습니다" 대신 "~에 도움이 될 수 있어요" |
| 혼내지 않지만 살짝 찔리게 | 죄책감 유발 아님, 자기 인식 유도 |
| 다음 행동 제안 | 결말은 항상 가능한 다음 행동으로 |

**예시 톤:**
> "이번 주 비타민C는 7회 중 4회 완료했어요. 완벽하진 않지만, 아예 손 놓은 주는 아니에요. 다음 주 목표는 거창하게 말고 5회. 팔은 아직 넣지 말고, 일단 입에 넣는 걸로 갑시다."

## 11. Progressive Onboarding 원칙

**핵심 원칙:**
> "최소 입력으로 바로 작동하고, 정보가 쌓일수록 더 똑똑해진다."

### 3단계 온보딩 구조

**1단계 Quick Start (필수)**
- 이름 또는 닉네임
- 관리하고 싶은 루틴 1개 (예: 비타민C 하루 1회, 매주 수/목 30분 운동)
- 이 단계만으로 앱 사용 즉시 시작 가능

**2단계 Health Context (선택)**
- 건강 관심사, 가족력, 현재 복용 영양제·약, 보험 보장 인지 여부
- 건너뛰기 가능
- 완료 시 미니 건강 리포트 즉시 생성

**3단계 Report Enrichment (선택)**
- 건강검진 내역, 상세 가족력, 병력, 상담 관심 영역
- 더 정밀한 건강 리포트 및 상담 품질 향상에 활용

### DB 설계 원칙 (Onboarding 연계)
- 모든 건강 관련 필드는 nullable 또는 선택 입력 가능
- `profile_completion_score` 필드로 입력 수준 추적
- 이 점수는 리포트 생성 품질 판단 및 CTA 노출 기준에 활용

## 12. 개인정보·동의·리드 관리 원칙

### 동의 관리

4가지 동의는 반드시 분리하여 관리합니다.

| 동의 항목 | 필수 여부 | 용도 |
|----------|----------|------|
| 개인정보 수집·이용 동의 | 필수 | 서비스 이용 전반 |
| 민감정보 처리 동의 | 필수 (건강 정보 입력 전) | 건강 관련 데이터 처리 |
| 제3자 제공 동의 | 필수 (상담 신청 전) | 보장 점검 상담 연결 |
| 마케팅 수신 동의 | 선택 | 마케팅 커뮤니케이션 |

**`consents` 테이블 운영 원칙:**
`consents`는 append-only 방식으로 관리합니다. 기존 동의 기록은 법적 증빙을 위해 수정하지 않습니다. 동의 철회 시 기존 row를 수정하는 대신 새로운 철회 이벤트 또는 `withdrawn_at` 기록을 남깁니다.

### 상담 리드 관리 원칙

- 상담 리드는 사용자의 동의 범위와 보유기간 내에서만 별도 관리합니다.
- 회원 탈퇴 또는 삭제 요청 시 정책에 따라 삭제 또는 익명화 처리합니다.
- `consultation_leads.user_id`는 계정 삭제 이후에도 법적·운영상 필요한 기간 동안 상담 이력 식별을 위해 nullable로 설계하되, 개인정보 보유기간과 삭제 정책을 따릅니다.

### 개인정보 보호

- PIPA(개인정보보호법) 준수: 민감정보 분리 저장·별도 동의
- 암호화: pgcrypto (AES-256 이상) — 건강 프로필·가족력·리드 연락처
- 전송 구간: TLS 1.3 이상
- 삭제 요청: 30일 이내 처리

## 14. DB 설계 원칙

### 핵심 원칙

1. **모든 건강 관련 필드는 nullable** — Progressive Onboarding 원칙 반영
